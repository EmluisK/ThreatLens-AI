## Discussion

The team met to coordinate the final feature addition: IoT malware classification using a LightGBM model trained on the CIC-YNU-IoTMal dataset.

EK reviewed the multi-modal classification approach with the team, covering how behavioral features from pcap captures, SAR reports, and strace traces are fused into a single feature vector. The team aligned on how IoT-sourced alerts would fit into the existing alert model and agreed on the severity mapping per malware family before implementation began.

AD implemented the iot_classifier.py service responsible for loading the LightGBM model from the ml_artifacts directory at startup and running inference through the classify function. The Alert model was updated to add the malware_family column and make log_id nullable so IoT alerts could be stored without a linked log file. Idempotent ALTER TABLE migrations were added to main.py to apply schema changes automatically on startup.

EI implemented the POST /ingest/iot endpoint that accepts a feature vector, runs classification, and creates an Alert with the predicted family and severity. The GET /ingest/iot/status endpoint was also added. The severity mapping logic was implemented to derive alert severity from the predicted family and confidence score. The admin audit log response was updated to include the malware_family field.

SD executed the full training pipeline on the CIC-YNU-IoTMal dataset, running data cleaning, zero-variance feature removal, gain-based feature selection, and training the LightGBM fusion model across all four architectures. Classification performance was validated across all five malware families on the held-out test set. The model artifacts were exported and verified against the live backend.

FH built the IoT Classifier tab in the Admin dashboard including a model status indicator, a JSON feature payload textarea, a submit button, and a results panel showing the predicted family, confidence score, and per-class probability bars. A FamilyBadge component was added and the Audit Log table was updated with a FAMILY column using distinct color coding per family. The Analyst dashboard was updated to surface the malware_family field on IoT-sourced alerts.

GZ added the IoT Classifier navigation link to the DashboardLayout under the admin section and confirmed the new tab integrates cleanly with the existing role-based navigation.

## Decisions

- IoT alert severity mapping confirmed: Mirai and DarkNexus map to critical, Gafgyt and Generic to high, Benign to low
- If confidence is below 50 percent, severity is capped at medium regardless of family
- Model artifacts are not included in the repository and must be generated by running the notebook
- Schema migrations run automatically on backend startup using idempotent ALTER TABLE statements

## Tasks Assigned

No new feature tasks were assigned. The team moved to testing, documentation, and final review.
