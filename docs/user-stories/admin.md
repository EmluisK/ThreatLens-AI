# User Stories - Security Admin

---

**US-A01 - Log In**
As a Security Admin, I want to log in with my email and password so that I can access the platform securely.

---

**US-A02 - Create User Accounts**
As a Security Admin, I want to register new user accounts and assign them a role so that I can control who has access to the platform and at what level.

---

**US-A03 - View All Users**
As a Security Admin, I want to see a list of all registered users so that I can keep track of who has access to the platform.

---

**US-A04 - Delete a User**
As a Security Admin, I want to remove a user account so that I can revoke access when someone leaves the team or no longer needs it.

---

**US-A05 - Upload Log Files**
As a Security Admin, I want to upload a log file and specify its source so that the platform can analyze it and generate alerts automatically.

---

**US-A06 - View Uploaded Logs**
As a Security Admin, I want to see all log files that have been uploaded so that I know what data the platform is working with.

---

**US-A07 - View the Audit Log**
As a Security Admin, I want to see all alerts across the platform including their severity, status, and source so that I have a full picture of everything that has been flagged.

---

**US-A08 - Submit an IoT Sample for Classification**
As a Security Admin, I want to submit a behavioral feature vector from an IoT binary so that the platform can classify it and generate an alert with the predicted malware family.

---

**US-A09 - Check IoT Classifier Status**
As a Security Admin, I want to see whether the IoT classification model is loaded and ready so that I know if submitted samples will be classified or queued without a result.

---

**US-A10 - View IoT Classification Results**
As a Security Admin, I want to see the predicted malware family, confidence score, and per-class probabilities for a submitted IoT sample so that I can assess the threat level before acting on it.
